# Infraスカウトシステム Spec

## 目的
採用担当1人でInfraのスカウト業務を効率化する。
スカウト選定・文章生成・送信・日程調整の手作業をAIで自動化する。

## 制約条件
- スカウトチケット：40枚（期限 2026/07/10）
- 1日の送信上限：2人
- 採用目標：総合職1名
- 勤務地：大阪/京橋 または 東京/世田谷区

## 現在の手作業フロー
1. 保存済み検索条件で学生一覧を取得
2. 1人ずつ手動でプロフィールを確認
3. 良さそうな人にスカウト文を手動で送信
4. 返信が来たら手動でやり取り
5. 面談日程を手動で調整

## 自動化する範囲
1. 検索結果の学生をペルソナと照合してスコアリング
2. スコアが高い学生のスカウト文を自動生成
3. 1日2人分を自動送信（InfraのAPIを使用）
4. 返信が来たらBookingsリンクを自動送付

## 自動化しない範囲
- Infraサイト内の検索条件の設定（既存の保存条件を使用）
- 最終的な送信判断（Dry Runで確認してから本番実行）

## スコアリング基準
### 必須条件（満たさない場合は除外）
- 稼働：週4以上（32h）または週5（40h）
- 勤務地：大阪/京橋 または 東京/世田谷区に通える

### 加点条件
- 自己PRに「自走」「主体的」「独学」などのキーワードがある
- 理系学部または情報系専攻
- 長期継続意欲がある

### 減点条件
- 「教えてもらいたい」「成長したい」だけで実績がない
- 週32h未満の稼働しかできない

## 使用ツール
- InfraのAPI（APIキー取得済み）
- Microsoft Bookings（面談日程調整）
- Googleスプレッドシート（応募者管理）
